# Data Model Summary

> Read this when working on database models, migrations, or API schemas.

Condensed model reference. For full field lists, see `src/humpback/database.py`.

- **ModelConfig** (`model_configs`) — ML model registry entry (name, path, vector_dim, model_type, input_format, is_default). `TFLiteModelConfig` is a backward-compatible alias.
- **AudioFile** (`audio_files`) — retained local-audio reference row (filename, folder_path, source_folder, checksum_sha256, duration_seconds, sample_rate_original). Used by local detection/call-parsing paths; the old Audio page and `audio_metadata` table are retired.
- **ClusteringJob** (`clustering_jobs`) — retained Vocalization / Clustering run (detection_job_ids JSON, parameters JSON, metrics_json, refined_from_job_id, error_message)
- **Cluster** (`clusters`) — one per cluster label per job (clustering_job_id FK, cluster_label, size, metadata_summary JSON)
- **ClusterAssignment** (`cluster_assignments`) — links a cluster to a retained source row (`cluster_id`, `source_id`, `embedding_row_index`)
- **ClassifierModel** (`classifier_models`) — binary classifier artifact (name, model_path .joblib, model_version, vector_dim, training_summary JSON, `training_source_mode`, optional `source_candidate_id`/`source_model_id`, `promotion_provenance` JSON for candidate-backed models)
- **ClassifierTrainingJob** (`classifier_training_jobs`) — training run (detection-job-backed or candidate-backed; persists `source_mode`, `source_detection_job_ids`, optional `source_candidate_id`/`source_model_id`, `manifest_path`, `training_split_name`, `promoted_config`, `source_comparison_context`, and `legacy_source_summary` for pre-cleanup jobs)
- **AutoresearchCandidate** (`autoresearch_candidates`) — imported autoresearch bundle for review/promotion (artifact paths, promoted config JSON, split metrics/deltas, source counts, warnings, source model metadata, exact-replay status, optional linked training/model ids)
- **DetectionJob** (`detection_jobs`) — local or hydrophone detection scan (classifier_model_id FK, audio_folder, confidence/hop/threshold params, detection_mode, window_selection, min_prominence, max_logit_drop, output_row_store_path, result_summary JSON, extract_* columns)
- **VocalizationLabel** (`vocalization_labels`) — per-detection vocalization type label (detection_job_id, row_id, label, source). Linked to detection rows by stable UUID `row_id`.
- **RetrainWorkflow** (`retrain_workflows`) — legacy classifier retrain history. New creation is retired, but historical rows remain listable/readable.
- **VocalizationType** (`vocalization_types`) — managed vocabulary entry for vocalization type classification (name, description, unique name constraint)
- **VocalizationClassifierModel** (`vocalization_models`) — multi-label vocalization model artifact (name, model_dir_path, vocabulary_snapshot JSON, per_class_thresholds JSON, per_class_metrics JSON, is_active)
- **VocalizationTrainingJob** (`vocalization_training_jobs`) — multi-label training run (source_config JSON with retained `detection_job_ids`; legacy embedding-set provenance may be archived into `archived_legacy_sources`, parameters JSON, vocalization_model_id set on completion)
- **VocalizationInferenceJob** (`vocalization_inference_jobs`) — scoring run (vocalization_model_id FK, source_type, source_id, output_path to predictions parquet)
- **DetectionEmbeddingJob** (`detection_embedding_jobs`) — post-hoc detection embedding generation and sync (detection_job_id, model_version, composite unique on (detection_job_id, model_version), mode [full/sync], progress_current/total, rows_processed/rows_total, result_summary JSON, status)
- **TrainingDataset** (`training_datasets`) — unified editable snapshot of training embeddings and labels (name, source_config JSON, parquet_path, total_rows, vocabulary JSON). Active sources are detection-job-based; archived legacy source metadata may remain in `source_config`.
- **TrainingDatasetLabel** (`training_dataset_labels`) — per-row label on a training dataset (training_dataset_id FK, row_index, label, source). Indexed on (training_dataset_id, row_index).
- **HyperparameterManifest** (`hyperparameter_manifests`) — data manifest for hyperparameter search (name, status, embedding_model_version, training_job_ids JSON, detection_job_ids JSON, split_ratio JSON, seed, manifest_path, example_count, split_summary JSON, detection_job_summaries JSON, error_message, completed_at)
- **HyperparameterSearchJob** (`hyperparameter_search_jobs`) — random search over classifier hyperparameters (name, status, manifest_id FK, search_space JSON, n_trials, seed, objective_name, results_dir, trials_completed, best_objective, best_config JSON, best_metrics JSON, comparison_model_id FK, comparison_threshold, comparison_result JSON, error_message, completed_at)
- **CallParsingRun** (`call_parsing_runs`) — parent row threading one four-pass call parsing pipeline run (status, `audio_file_id` / `hydrophone_id` / `start_timestamp` / `end_timestamp` exactly-one-of source identity, `config_snapshot` TEXT, `region_detection_job_id` / `event_segmentation_job_id` / `event_classification_job_id` child FKs, error_message, completed_at)
- **RegionDetectionJob** (`region_detection_jobs`) — Pass 1 region detection job (status, parent_run_id, `audio_file_id` / `hydrophone_id` / `start_timestamp` / `end_timestamp` source, model_config_id, classifier_model_id, `config_json` TEXT with `RegionDetectionConfig`, trace_row_count, region_count, started_at, completed_at, error_message). Writes `trace.parquet` + `regions.parquet` under `storage_root/call_parsing/regions/<job_id>/`.
- **EventSegmentationJob** (`event_segmentation_jobs`) — Pass 2 framewise segmentation job (status, parent_run_id, region_detection_job_id FK, segmentation_model_id, config_json with `SegmentationDecoderConfig`, event_count, compute_device, gpu_fallback_reason, started_at, completed_at, error_message). Worker loads CRNN checkpoint, resolves audio source from upstream Pass 1 job, validates the inference device once at load time (MPS/CUDA → CPU on validation failure), runs per-region inference, writes `events.parquet` under `storage_root/call_parsing/segmentation/<job_id>/`. `compute_device` is `"mps"`, `"cuda"`, or `"cpu"` (NULL on pre-migration rows); `gpu_fallback_reason` is non-NULL only when GPU was attempted and rejected (e.g. `"mps_output_mismatch"`, `"cuda_load_error"`).
- **EventClassificationJob** (`event_classification_jobs`) — Pass 3 per-event multi-label classification job (status, parent_run_id, event_segmentation_job_id FK, vocalization_model_id, config_json, typed_event_count, compute_device, gpu_fallback_reason, started_at, completed_at, error_message). Worker resolves audio source transitively through Pass 2/Pass 1, validates the inference device at load time, classifies each event crop, and writes `typed_events.parquet`. `compute_device` and `gpu_fallback_reason` follow the same conventions as Pass 2.
- **SegmentationModel** (`segmentation_models`) — Pass 2 PyTorch segmentation model registry (name, model_family, model_path, config_json, training_job_id, created_at). Distinct from `vocalization_models` because framewise segmentation is a different task type.
- **SegmentationTrainingDataset** (`segmentation_training_datasets`) — named container for Pass 2 training samples (name, description). Shared by the bootstrap script and the future UI timeline-editor workflow.
- **SegmentationTrainingSample** (`segmentation_training_samples`) — one audio crop with framewise event annotations (training_dataset_id FK, `audio_file_id` / `hydrophone_id` / `start_timestamp` / `end_timestamp` source identity, crop_start_sec, crop_end_sec, events_json with `[{"start_sec", "end_sec"}]` audio-relative bounds, source, source_ref for idempotency). Indexed on training_dataset_id and composite `(training_dataset_id, source_ref)`.
- **SegmentationTrainingJob** (`segmentation_training_jobs`) — queued Pass 2 trainer run (status, training_dataset_id FK, config_json with `SegmentationTrainingConfig`, segmentation_model_id set on completion, result_summary JSON, error_message, started_at, completed_at).
- **WindowClassificationJob** (`window_classification_jobs`) — standalone sidecar scoring cached Pass 1 Perch embeddings through vocalization classifiers (status, region_detection_job_id FK, vocalization_model_id FK, config_json, window_count, vocabulary_snapshot JSON array, error_message, started_at, completed_at). Writes `window_scores.parquet` (wide format: time_sec, region_id, one float64 column per vocalization type) under `storage_root/call_parsing/window_classification/<job_id>/`.
- **WindowScoreCorrection** (`window_score_corrections`) — per-window label override for window classification review (window_classification_job_id FK, time_sec, region_id, correction_type "add"/"remove", type_name). Multiple corrections per window are valid (multi-label).
- **ContinuousEmbeddingJob** (`continuous_embedding_jobs`) — Sequence Models producer; one row per encoding signature, dispatched on `model_version` family (ADR-057). Common: status, model_version, target_sample_rate, feature_config_json, encoding_signature for idempotency, vector_dim, parquet_path, error_message. SurfPerch source (nullable post-061): event_segmentation_job_id FK, window_size_seconds, hop_seconds, pad_seconds, total_events, merged_spans, total_windows. CRNN region-based source (nullable, added in 061): region_detection_job_id FK, event_segmentation_job_id FK (Pass 2 disambiguator), crnn_segmentation_model_id FK, crnn_checkpoint_sha256, chunk_size_seconds, chunk_hop_seconds, projection_kind, projection_dim, total_regions, total_chunks.
- **HMMSequenceJob** (`hmm_sequence_jobs`) — Sequence Models HMM trainer (status, continuous_embedding_job_id FK, event_classification_job_id FK (nullable in storage but required after submit per ADR-063, migration 066; ON DELETE RESTRICT), n_states, pca_dims, pca_whiten, l2_normalize, covariance_type, n_iter, random_seed, min_sequence_length_frames, tol, library, train_log_likelihood, n_train_sequences, n_train_frames, n_decoded_sequences, artifact_dir, error_message). CRNN-only training-mode + tier configuration (nullable, added in 061): training_mode (`full_region` | `event_balanced` | `event_only`), event_core_overlap_threshold, near_event_window_seconds, event_balanced_proportions (JSON), subsequence_length_chunks, subsequence_stride_chunks, target_train_chunks, min_region_length_seconds. Writes `pca_model.joblib`, `hmm_model.joblib`, `decoded.parquet` (renamed from `states.parquet` in migration 063; column `state` renamed to `label` for cross-source naming parity per ADR-061; HMM API serialization layer maps `label` back to `viterbi_state` for API stability), `transition_matrix.npy`, `state_summary.json`, `training_log.json` under `storage_root/hmm_sequences/<job_id>/`. A read-time loader shim falls back to the legacy `states.parquet`/`state` shape on pre-migration job dirs.
- **MaskedTransformerJob** (`masked_transformer_jobs`, ADR-061, migration 063; retrieval-head config migration 067; sequence-construction config migration 068; contrastive config migration 069; batch-size config migration 070) — Sequence Models masked-span transformer trainer parallel to HMM (status, status_reason, continuous_embedding_job_id FK, event_classification_job_id FK (nullable in storage but required after submit per ADR-063, migration 066; ON DELETE RESTRICT), training_signature unique-indexed, training config: preset, mask_fraction, span_length_min, span_length_max, dropout, mask_weight_bias, cosine_loss_weight, batch_size, retrieval_head_enabled, retrieval_dim, retrieval_hidden_dim, retrieval_l2_normalize, sequence_construction_mode, event_centered_fraction, pre_event_context_sec, post_event_context_sec, contrastive_loss_weight, contrastive_temperature, contrastive_label_source, contrastive_min_events_per_label, contrastive_min_regions_per_label, require_cross_region_positive, related_label_policy_json, max_epochs, early_stop_patience, val_split, seed; tokenization config: k_values JSON; device + outcomes: chosen_device, fallback_reason, final_train_loss, final_val_loss, total_epochs; storage: job_dir, total_sequences, total_chunks; UTC timestamps). `sequence_construction_mode` is `"region"` (default), `"event_centered"`, or `"mixed"`; existing rows backfill to region mode with `event_centered_fraction=0.0` and null context fields. Event-centered and mixed modes use `load_effective_events()` with default 2.0s pre/post context to construct training windows only; extraction artifacts remain full-region aligned. Contrastive fields backfill to disabled behavior (`contrastive_loss_weight=0.0`, `contrastive_label_source="none"`); enabled jobs require event-centered or mixed sequence construction, use human `VocalizationCorrection` labels only, and exclude model Classify labels from contrastive positives. `batch_size` backfills to `8`. `training_signature` excludes `k_values` so a completed job can extend its k-sweep without retraining; non-default `batch_size` participates; retrieval-head fields participate when `retrieval_head_enabled=true`; event-centered/mixed sequence-construction fields participate when non-region; positive contrastive config participates and includes the resolved Classify workflow id; default region/disabled rows preserve prior signature behavior. Restricted at v1 to upstream `continuous_embedding_jobs` rows with `model_version="crnn-call-parsing-pytorch"` (CRNN region-based source). Writes `transformer.pt`, `contextual_embeddings.parquet`, optional `retrieval_embeddings.parquet` for retrieval-head jobs, `loss_curve.json`, `reconstruction_error.parquet`, plus per-k subdirs `k<N>/{decoded.parquet, kmeans.joblib, overlay.parquet, exemplars.json, label_distribution.json, run_lengths.json}` under `storage_root/masked_transformer_jobs/<job_id>/`. For retrieval-head jobs the per-k tokenizer is fit from `retrieval_embeddings.parquet`; old and retrieval-disabled jobs continue to tokenize contextual embeddings. Per-k subdirs are written atomically via `k<N>.tmp/` → rename.
- **MotifExtractionJob** (`motif_extraction_jobs`) — Sequence Models motif miner. Consumes a completed parent (HMM job or masked-transformer job + k), collapses repeated label sequences, mines symbolic n-grams, writes `manifest.json`, `motifs.parquet`, and `occurrences.parquet` under `storage_root/motif_extractions/<job_id>/`. Columns: status, parent_kind (`"hmm"` | `"masked_transformer"`, NOT NULL, added in migration 064), nullable hmm_sequence_job_id FK, nullable masked_transformer_job_id FK, nullable k (parent-kind-conditional), source_kind, min_ngram, max_ngram, minimum_occurrences, minimum_event_sources, rank weights, optional call_probability_weight, config_signature (includes parent_kind + parent FK + k for parent-aware idempotency), counters, artifact_dir, error_message. CHECK constraint enforces XOR between parent FKs and consistency with `parent_kind`; `k IS NOT NULL` iff `parent_kind="masked_transformer"`. Existing pre-migration rows backfill with `parent_kind="hmm"`.
